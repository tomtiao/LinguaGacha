# Product

## Register

product

## Users

LinguaGacha 面向需要批量翻译小说、游戏文本、字幕、电子书和 Markdown 等内容的创作者、汉化者、翻译协作者与个人用户。用户通常希望用尽量少的配置完成高质量翻译，并在翻译过程中保持术语一致、格式稳定和结果可校对。

## Product Purpose

LinguaGacha 是一个桌面端 AI 文本翻译工具，目标是在复杂文本工程中提供开箱即用、速度快、质量稳定的翻译工作流。它应降低用户处理模型配置、术语表、文本保护、格式保留和校对修正的成本，让用户更专注于作品本身，而不是被工具流程拖住。

## Brand Personality

萌、极客、内敛。界面可以有轻微的可爱气质和技术感，但表达应克制，像一个可靠又有点俏皮的原生桌面工具，而不是高调张扬的网页产品。

## Anti-references

避免“网页感”：整体视觉、布局节奏和交互反馈都应像一款原生桌面客户端，而不是浏览器网页或 Web SaaS 套壳。避免网页式大留白、滚动长页、营销式 hero、卡片堆叠的信息流、过强的视觉宣传感，以及像后台管理模板一样的通用 Web 控件气质。界面应更强调桌面应用的稳定壳层、明确工具区、紧凑信息密度、可重复操作路径和即时反馈。

## Design Principles

1. 原生桌面客户端优先：界面结构、交互节奏和状态反馈应让用户感到这是一个稳定的本地工具，而不是网页页面。
2. 工具效率优先：首屏和主要页面应直接服务翻译、配置、校对和文件处理，不制造网页式浏览动线。
3. 复杂能力轻量呈现：把模型、规则、术语、文本保护等强功能组织成稳定、可扫描、可重复操作的工具界面。
4. 可爱但不吵闹：萌感来自细节、节奏、文案和微交互，而不是大面积装饰或主题化表演。
5. 极客但不排外：保留高级用户需要的控制力，同时让新用户能顺着流程完成任务。
6. 结果可信：所有涉及翻译状态、任务进度、错误、覆盖写入和校对修改的反馈都要明确、可追踪。

## Accessibility & Inclusion

默认遵循 WCAG AA 级别的可读性与交互可达性；支持键盘可操作、清晰焦点态、足够颜色对比，并尊重减少动态效果等系统偏好。
